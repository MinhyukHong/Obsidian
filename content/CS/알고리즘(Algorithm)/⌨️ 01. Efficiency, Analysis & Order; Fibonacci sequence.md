
## 피보나치 수 구하기 - 재귀적 방법
---
문제: $n$번째 피보나치 수를 구하라
입력: 양수 $n$
출력: $n$번째 피보나치 수

> *Pseudo Code*
```
int fib(int n) {
	if(n <= 1)
		return n;
	else
		return fib(n-1) + fib(n-2);
}
```



## Discussion
---
- 피보나치 수 구하기 **재귀 알고리즘**은 수행 속도가 매우 느리다.
	- 이유: 같은 피보나치 수를 중복 계산
	- e.g., `fib(5)` 계산을 위해 `fib(2)` 3번 중복 계산

## `fib(n)`의 함수 호출 횟수 계산
---
$T(n)=fib(n)$을 계산하기 위하여 $fib$ 함수를 호출하는 횟수. 즉, ==재귀 트리 상의 마디의 개수==

$T(0)=1$
$T(1)=1$
$T(n)=T(n-1)+T(n-2)+1$
	$> 2*T(n-2)$
	$> 2^2*T(n-4)$
	$> 2^3*T(n-6)$
왜냐하면, 각각 $T(n-1)>T(n-2)$, $T(n-2)>2*T(n-4)...$
	$>2^{n/2}*T(0)$ 
	$=2^{n/2}$

$n$이 홀수일 때에도 유사하게 증명이 가능하다.


## 계산한 호출 횟수의 검증
---
> 정리: 재귀적 알고리즘으로 구성한 재귀 트리의 마디의 수를 $T(n)$이라고 하면, $n\ge2$인 모든 $n$에 대하여 $T(n)>2^{n/2}$이다.

- ==증명==: ($n$에 대한 수학적 귀납법으로 증명)
- 귀납 출발점:
	$T(2)=T(1)+T(0)+1=3>2=2^{2/2}$
	$T(3)=T(2)+T(1)+1=5>2.83\approx2^{3/2}$

- 귀납 가정: $2\le m<n$인 모든 $m$에 대해서 $T(m)>2^{m/2}$이라 가정
- 귀납 절차: $T(n)>2^{n/2}$ 임을 보이면 된다.
	$>2^{(n-1)/2}+2^{(n-2)/2}+1$
	$>2^{(n-2)/2}+2^{(n-2)/2}$
	$=2*2{(n/2)-1}$
	$=2^{n/2}$


## 피보나치 수 구하기 알고리즘 - 반복적 방법
---
>*Pseudo Code*

```
int fib2(int n) {
	index i;
	int f[0..n];
	f[0] = 0;
	if(n > 0) {
		f[1] = 1;
		for(i=2; i<=n; i++)
			f[i] = f[i-1] + f[i-2];
		}
	return f[n];
}
```


## Discussion
---
- **반복 알고리즘**은 수행속도가 훨씬 더 빠르다.
	- 이유: 중복 계산이 없다.
- 계산하는 항의 총 개수
	- $T(n)=n+1$
	- 즉, $f[0]$부터 $f[n]$까지 한 번 씩만 계산한다.
