
> 다음의 숫자를 오름차순으로 정렬하는 프로그램을 작성해보자.
> 1 10 5 8 7 6 4 3 2 9

버블 정렬은 선택 정렬과 같이 직관적인 해결 방법이다. 바로 가까이에 있는 두 숫자끼리 비교하여 더 작은 숫자를 앞으로 보내주는 것을 반복한다. 버블 정렬은 정렬 알고리즘 중에서 구현은 가장 쉬울지라도 가장 비효율적인 알고리즘이다.


> *옆에 있는 값과 비교해서 더 작은 값을 앞으로 보내면 어떨까?*

```
#include <stdio.h>

int main(void) {
	int i, j, temp;
	int array[10] = {1, 10, 5, 8, 7, 6, 4, 3, 2, 9};
	for(i = 0; i < 10; i++) {
		for(j = 0; j < 9 - i; j++){
			if(array[j] > array[j + 1]) {
				temp = array[j];
				array[j] = array[j + 1];
				array[j + 1] = temp;
			}
		}
	}
	for(i = 0; i < 10; i++) {
		printf("%d ", array[i]);
	}
	return 0;
} 
```

버블 정렬의 시간 복잡도는 $O(N^2)$ 이다. 선택 정렬과 동일하지만, 효율은 더 안 좋다. 자리를 바꾸는 총 세 개의 명령어를 사용하여 실제로 컴퓨터가 작업하는 양이 훨씬 많다.

버블 정렬은 단순히 반복적으로 두 숫자를 비교하여 앞으로 보낸다. 이 과정에서 각 싸이클마다 가장 큰 값이 맨 뒤로 보내지게 된다. 내부적인 컴퓨팅 연산이 가장 비효율적으로 일어나게 되어 실제 수행시간이 가장 느린 알고리즘이다.
