
## graph coloring
---
- 지도 칠하기(Map coloring): 인접하는 지역을 구분하기 위해 색을 할당하는 문제
- 4개의 색이면 충분하다.
- 어떤 지도, 영역의 구분일지라도 4개의 색이면 충분하다는 사실이 밝혀져 있다.
- graph coloring: 인접하는 노드에 같은 색깔이 할당되지 않도록 색을 할당하는 문제

## m-coloring problem
---
- 지도에 m가지 색으로 색칠하는 문제
	- m개의 색을 가지고, 인접한 지역이 같은 색이 되지 않도록 지도에 색칠하는 문제

![Figure 2](Backtracking_gc_3.png)

- 위 그래프에서 두 가지 색으로 문제를 풀기는 불가능하다.
- 세 가지 색을 사용하면 총 6가지의 해답을 얻을 수 있다.


## 그래프 색칠하기 되추적 해법
---
>*Pseudo Code*
```
void m_coloring(index i){
	int color;
	
	if(promising(i))
		if(i==n)
			cout << vcolor[1] through vcolor[n];
		else
			for(color=1; color<=m; color++){
				vcolor[i+1] = color;
				m_coloring(i+1);
			}
}

bool promising(index i){
	index j;
	bool switch;
	
	switch = true;
	j=1;
	while(j<i && switch){
		if(W[i][j] && vcolor[i]==vcolor[j]) // W[i][j]: 연결표시. T or F
			switch = false;
		j++;
	}
	return switch;
}
```


## 분석
---
- m개의 색. n개의 정점을 가진 비방향그래프
- 상태공간트리 상의 마디의 총수

$$
1+m+m^2+...+m^n=m^{n+1}-1/{m-1}
$$

- 최악의 경우: n에 도달해서야 solution이 없다는 것을 알게 된다.
	- m=2일 경우
	- 거의 모든 경우를 생성하지 않고는 답이 없다는 것을 알 수 없다.
