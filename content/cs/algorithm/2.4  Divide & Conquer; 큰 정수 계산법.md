
## 큰 정수 곱셈
---
- 문제: 2개의 큰 정수 u와 v의 곱
- 입력: 큰 정수 u와 v, 크기 n
- 출력: prod(u와 v의 곱)

> *Pseudo Code*
```
large_integer prod(large_integer u, large_integer v) {
	large_integer x, y, w, z;
	int n, m;
	
	n = maximum(u의 자리수, v의 자리수);
	if(u == 0 || v == 0) return 0;
	else if(n <= threshold)
		return 일반적인 방법으로 구한 u x v;
	else {
		m = floor(n/2);
		x = u divide 10^m; y = u mod 10^m;
		w = v divide 10^m; z = v mod 10^m;
		return prod(x, w) x 10^2m + (prod(x, z)+prod(w, y)) x 10^m + prod(y, z);
	}
}
```


## 시간복잡도 분석 - prod 최악의 경우
---
- 단위 연산: 덧셈, 뺄셈, divide $10^m$, mod $10^m$, $*10^m$
- 입력 크기: 정수의 자리수, n
- n이 2의 거듭제곱 형태라고 가정
- 덧셈, 뺄셈, divide $10^m$, mod $10^m$, $*10^m$에 있는 1차시간 연산은 모두 $cn$으로 표시.
- $W(n)=4W(\frac {n}{2})+cn, n>s$이고, n이 2의 거듭제곱인 경우
- $W(s)=0$
- $W(n)\in \Theta(n^{lg4})=\Theta(n^2)$


### 개선된 방법
---
- 이전 방법에서는 $xw, xz+yw, yz$의 계산 필요 - ==4회의 곱셈==
- 개선 방법: $r$ 계산 추가
- $r=(x+y) *(w+z)=xw+(xz+yw)+yz$
	- $xz+yw = r-xw-yz$
- 결과적으로 $xw, xz+yw, yz$를 계산하는데, 덧셈과 뺄셈의 횟수는 증가하지만, 곱셈은 3회가 필요하다.


## 큰 정수 곱셈 Revision
---
- 문제: 2개의 큰 정수 u와 v의 곱
- 입력: 큰 정수 u와 v, 크기 n
- 출력: prod(u와 v의 곱)

> *Pseudo Code*
```
large_integer prod2(large_integer u, large_integer v) {
	large_integer x, y, w, z, r, p, q;
	int n, m;
	n = maximum(u의 자리수, v의 자리수);
	if(u == 0 || v == 0) return 0;
	else if(n <= threshold)
		return 일반적인 방법으로 구한 u x v;
	else {
		m = floor(n/2);
		x = u divide 10^m; y = u mod 10^m;
		w = v divide 10^m; z = v mod 10^m;
		r = prod2(x+y, w+z);
		p = prod2(x, w);
		q = prod2(y, z);
		return p x 10^2m + (r-p-q) x 10^m + q;
}
```


## 시간복잡도 분석 - prod2 최악의 경우
---
- $prod2(x+y, w+z), n/2\le입력크기\le n/2+1$
- $prod2(x, w), n/2$
- $prod2(y,z), n/2$
- $3W(\frac {n}{2})+cn\le W(n)\le 3W(\frac{n}{2}+1)+cn, n>s$ 이고, $n$이 2의 거듭제곱인 경우(cn은 분할과 결합에 드는 선형 시간),
	- $W(s)=0$
	- $W(n)\in \Theta(n^{lg3}) \approx \Theta(n^{1.58})$
