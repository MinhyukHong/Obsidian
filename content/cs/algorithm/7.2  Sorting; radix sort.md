
## 분배에 의한 정렬: 기수정렬
---
- 키에 대해서 아무런 정보가 없는 경우
	- 키들을 비교하는 것 이외에는 다른 방법이 없으므로 $\Theta(nlgn)$ 보다 더 좋은 알고리즘을 만드는 것은 불가능하다.
- 키에 대한 어느 정도의 정보를 알고 있는 경우
	- digit의 개수가 모두 같다면, 첫 번째 digit가 같은 수끼리 따로 모으고, 그 중에서 두 번째 digit가 같은 수끼리 따로 모으고, 마지막 digit까지 이런 식으로 계속 모으는 방법으로 각 digit를 한 번씩만 조사를 하면 정렬을 완료할 수 있다.
	- 분배에 의한 정렬(sorting by distribution) - 기수정렬(radix sort)
	- 기수(radix, base)를 사용

## 기수정렬(왼쪽에서 오른쪽 자리 순으로)
---
![Figure 1](sorting_radix_1.png)

- 왼쪽에서 오른쪽 순으로 하는 경우 뭉치(pile)를 구성하는 개수가 항상 일정하지 않으므로 관리하기가 쉽지 않다. 이를 해결하기 위해서는 다음 예와 같이 끝에 있는 digit부터 먼저 조사를 시작하면 된다.

![Figure 1](sorting_radix_2.png)

>*Pseudo Code*
```
void radixsort(node_pointer& masterlist, int numdigits){
	index i;
	node_pointer list[0..9];
	
	// numdigits
	for(i=1; i<=numdigits; i++){
		distribute(masterlist,i);
		coalesce(masterlist);
	}
}

void distribute(node_pointer& masterlist, index i){
	index j;
	node_pointer p;
	for(j=0; j<=9; j++)
		list[j]=NULL;
	p = masterlist;
	
	// n
	while(p!=NULL){
		j=p->key에서(오른쪽에서) i번째 숫자의 값;
		p를 list[j]의 끝에 링크;
		p = p->link;
	}
}

void coalesce(node_pointer& masterlist){
	index j;
	masterlist = NULL;
	
	// 10
	for(j=0; j<=9; j++)
		list[j]에 있는 마디들을 masterlist의 끝에 링크
}
```


## 알고리즘 분석
---
- 단위연산: 뭉치에 수를 추가하는 연산
- 입력크기: 정렬하는 정수의 개수 = n, 각 정수를 이루는 digit의 최대 개수 = numdigits
- 모든 경우 ==시간복잡도 분석==:
	$$
	T(n)=numdigits*(n+10)\in \Theta(numdigits*n)
	$$
	따라서 numdigits가 n과 같으면, 시간복잡도는 $\Theta(n^2)$ 가 된다. 그러나 일반적으로 서로 다른 n개의 수가 있을 때 그것을 표현하는데 필요한 digit의 수는 $lgn$ 으로 볼 수 있다.
- ==공간복잡도 분석==
	- 추가적으로 필요한 공간은 키를 연결된 리스트로 만드는데 필요한 공간(link의 공간), $\Theta(n)$

> 일반적인 기수정렬의 시간복잡도:
> $T(n)=d*(n+k)$
> d: digit numbers(자리수)
> n: 정렬할 data 수
> k: 각 자리의 범위(2진수: 2(0 or 1), 10진수: 10(0~9))

$T(n)\in \Theta(dn+dk)$
